"# Post Publisher - Публікація постів

Веб-застосунок для публікації, редагування та управління постами з фронтенд-валідацією форм та збереженням даних у локальному сховищі браузера.

     - Перетягніть файл `index.html` в вікно браузера, або
     - Введіть у адресному рядку: `file:///C:/Users/KM/Downloads/my-js/index.html`

2. **За допомогою локального веб-сервера:**
   ```bash
   # Якщо встановлений Python 3:
   cd c:/Users/KM/Downloads/my-js
   python -m http.server 8000
   # Потім відкрийте http://localhost:8000
   ```

3. **За допомогою Live Server в VS Code:**
   - Встановіть розширення "Live Server"
   - Клацніть правою кнопкою на `index.html`
   - Виберіть "Open with Live Server"

##  Функціональність

### Реалізовані можливості:

-  **Форма для створення постів** з полями:
  - Вибір юзера (спадаючий список)
  - Заголовок
  - Категорія (News, Tutorial, Opinion, Announcement)
  - Текст (максимум 200 слів)
  - Автор (email)

- **CRUD операції:**
  - Створення нових постів
  - Редагування існуючих постів
  - Видалення постів
  - Перегляд всіх постів у таблиці

-  **Зберігання даних:**
  - Всі посты зберігаються у `localStorage` браузера
  - Дані зберігаються автоматично при додаванні/редагуванні
  - Дані персистуються при перезавантаженні сторінки

-  **Зручні функції:**
  - Автоматичне заповнення email при виборі юзера зі списку
  - Счетчик слів в реальному часі (рахує та обрізає до 200 слів)
  - Форматування дат у українській локалізації
  - Безпека: HTML-екранування для запобігання XSS
  - Скорочення довгих текстів в таблиці

##  Методи валідації

### 1. **Валідація на рівні поля (Real-time validation):**
   - **На вході (`input` event):**
     - Заголовок: перевірка наявності тексту
     - Текст: підрахунок слів та обрізання до 200 слів
     - Email: перевірка формату
   
   - **На розфокусуванні (`blur` event):**
     - Всі поля перевіряються знову для повної валідації

### 2. **Повна валідація всієї форми (`validateAll()`):**
   Викликається перед збереженням:
   - ✓ **Заголовок** - обов'язкове, не порожнє
   - ✓ **Категорія** - обов'язково обрано значення
   - ✓ **Текст** - обов'язкове, від 1 до 200 слів
   - ✓ **Email (Автор)** - обов'язкове, перевірка формату за regex: `/^\S+@\S+\.\S+$/`

### 3. **Конкретні валідаційні правила:**

| Поле | Правило | Повідомлення про помилку |
|------|---------|--------------------------|
| Заголовок | Не може бути порожним | "Поле обов'язкове" |
| Категорія | Повинна бути обрана | "Оберіть категорію" |
| Текст | 1-200 слів | "Поле обов'язкове" або "Перевищено ліміт 200 слів" |
| Email | Формат `xxx@xxx.xxx` | "Невірний email" |

### 4. **Мінус обробка помилок:**
   - Помилки відображаються під кожним полем в реальному часі
   - Невалідні поля відмічаються класом `invalid` (червоне виділення)
   - Помилки автоматично очищуються при коректному вводі

### 5. **Допоміжні функції валідації:**
   - `countWords(text)` - точно рахує слова
   - `validateField(fieldName)` - валідація окремого поля
   - `validateAll()` - повна валідація перед збереженням

##  Збереження даних

Посты зберігаються у `localStorage` з ключем `posts_data_v1` у форматі JSON:

```json
[
  {
    "id": "1708000000000123",
    "title": "Заголовок",
    "category": "News",
    "text": "Текст до 200 слів...",
    "author": "email@example.com",
    "createdAt": "2026-02-19T10:30:00.000Z"
  }
]
```

##  Тестування валідації

1. Спробуйте створити пост без заголовку - появиться помилка
2. Введіть невалідний email - появиться помилка
3. Узпишіть більше ніж 200 слів - текст автоматично обрізатиметься
4. Виберіть юзера - email заповниться автоматично
5. Редагуйте існуючий пост - дані завантажаться у форму
6. Видаліть пост - він буде видален з таблиці та localStorage
